## 2. Blocker closure

### BLOCKER 1 — immutable session identity

**CLOSED**

The browser now uses a `BrowserSessionState` generation model.

Each operation captures:

```text
sid
generation
```

The binding is retained through:

```text
recording
-> WAV finalization
-> HTTP upload
-> upload response
-> voice.asr send
-> projected events / ACK
-> MP3 fetch
-> playback replacement
```

A response is accepted only while its SID/generation is still current.

Session creation and selection are disabled in active states:

```text
recording
uploading
transcribing
thinking
synthesizing
attaching
creating
```

Late frames from older session subscriptions are ignored by SID. Late upload results cannot dispatch ASR into a newly selected session. Late MP3 responses cannot replace current playback.

Replay records the latest historical `speech.tts.chunk.audio_ref`, waits for `session.resumed`, then performs one fetch for the final selected artifact.

Deterministic Web unit tests prove:

- immutable operation binding;
- stale deferred upload rejection;
- current-create-only `session.created`;
- old-session frame rejection;
- stale MP3 response rejection;
- latest-artifact replay coalescing.

### BLOCKER 2 — bounded failure and recovery

**CLOSED**

The UI state machine now uses:

```text
failed
failed -> reset -> ready
```

Bounded provider-neutral messages exist for:

- ASR cancellation;
- route denial;
- invalid ASR input;
- generic request failure;
- TTS failure after successful visible text;
- projected error/route-denied events;
- bounded failure progress stages;
- upload failure;
- microphone failure;
- speech fetch/playback failure.

TTS failure preserves the transcript and assistant final.

Reset:

- invalidates the active operation generation;
- discards the recorder;
- stops/revokes playback;
- clears warning/timer state;
- preserves the selected canonical session;
- returns to `ready`;
- sends no cancel, interrupt, worker, provider, or canonical operation.

The served UI now contains:

- “30 seconds or less” guidance;
- 60-second automatic-stop statement;
- Reset/Retry;
- reload/replay session link.

### BLOCKER 3 — deterministic concurrent upload proof

**CLOSED**

`WebVoiceHttp` now exposes a constructor-only test seam:

```ts
afterUploadReservation?: (sid) => Promise<void>
```

The seam is rejected unless:

```text
NODE_ENV=test
or
CI=true
```

It is not reachable through YAML, CLI, environment configuration, query parameters, browser input, or production limits.

Case E now executes:

```text
first request owns upload reservation
-> deterministic barrier reached
-> second request returns 409
-> release first request
-> first request returns 201
-> exactly one artifact
```

The previous 20 ms scheduling sleep has been removed as synchronization.

## 3. Accepted architecture and trust surfaces

### Static Web

- exact static allowlist;
- native browser ESM;
- no framework or bundler;
- no remote resources;
- restrictive CSP;
- `no-store`, `nosniff`, and `no-referrer`;
- source-first runtime.

### Token and WebSocket

- token held in browser memory only;
- password input cleared after connect;
- HTTP Bearer token;
- existing WebSocket query-token connection;
- invalid token closes `4401` before operations;
- no new listener;
- no request-query logging.

### Recording and upload

- Web Audio PCM capture;
- deterministic mono resampling;
- 16 kHz mono PCM16;
- canonical 44-byte WAV;
- 960,000-sample hard cap;
- 1,920,044-byte body cap;
- bounded streaming reader;
- independent strict gateway parser;
- gateway-derived labels and region;
- same-content collision fail-closed.

### ASR, turn, and TTS

- existing R0.9-01 `voice.asr`;
- one `#submitVoiceFinalTranscript`;
- one `#acceptTurnInput`;
- one production TurnRunner;
- shared visible-final TTS helper;
- coordinator-owned MiMo/MiniMax provider execution;
- one canonical ASR final, turn, model request, TTS artifact, and TTS chunk in the success path.

### Projection and replay

- `surface=web-v0` is provider-neutral;
- default CLI/developer frames remain unchanged;
- no provider/model/worker/path/private evidence crosses the Web surface;
- canonical JSONL remains authoritative;
- reload reconstructs transcript, final, and audio reference through `session.attach`.

### MP3 ownership

Speech fetch requires:

- valid token;
- existing valid session;
- canonical `speech.tts.chunk.audio_ref` reference;
- exact registry record;
- speech kind;
- MPEG MIME;
- contained non-symlink regular file;
- stable file identity;
- exact size/hash/magic.

Wrong ownership and invalid artifacts collapse to `404`.
