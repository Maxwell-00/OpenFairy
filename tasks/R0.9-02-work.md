### HTTP status/error table

| Status | Fixed use |
|---|---|
| `200` | Static asset or fully validated owned MP3. |
| `201` | Valid canonical WAV registered and revalidated. |
| `400` | Invalid SID syntax, invalid Content-Length, truncated/layout/size-mismatched WAV, aborted body. |
| `401` | Missing/wrong Bearer token. |
| `404` | Unknown route/session/artifact, unowned artifact, or any MP3 registry/path/file/hash/MIME/magic failure. |
| `405` | Wrong method on an exact route. |
| `409` | Active turn/ASR, same-session upload, or incompatible same-content artifact collision. |
| `413` | Declared or observed body above `1,920,044` bytes. |
| `415` | Wrong Content-Type, any Content-Encoding, unsupported/noncanonical WAV encoding/magic. |
| `422` | Empty/odd/over-960000-sample PCM contract. |
| `503` | Voice disabled or required region governance unavailable. |

Errors contain only a fixed code and bounded generic message. MP3 existence/ownership failures are deliberately collapsed to `404`.

## Recording and WAV design

### Closed constants

| Constant | Value |
|---|---:|
| target sample rate | 16000 Hz |
| channels | 1 |
| bits per sample | 16 |
| recommended | 480000 samples / 30 s |
| warning | 800000 samples / 50 s |
| countdown | 880000 samples / 55 s |
| maximum | 960000 samples / 60 s |
| maximum WAV bytes | 1920044 |

Formulas:

```text
target_equivalent = floor(total_source_samples * 16000 / actual_context_sample_rate)
accepted_increment = min(increment, 960000 - accepted_target_samples)
elapsed_seconds = accepted_target_samples / 16000
remaining_seconds = (960000 - accepted_target_samples) / 16000
WAV bytes = 44 + target_samples * 2
```

`setTimeout` is not present in the recorder module. Sample count is authoritative. Auto-stop is latched and calls the same `finalizeAndSend()` method as pointer/Space release; target output is truncated to `<=960000`, and a later release sees the in-progress/finalized guard and cannot upload twice.

The UI keeps one in-memory hold latch and one recorder identity. If pointer/Space release occurs while microphone permission or AudioWorklet startup is still pending, startup immediately enters the same finalize path after it resolves instead of leaving an unattended recorder. Pointer cancellation, lost capture, socket close, and pagehide detach the recorder identity and discard it; a late startup completion detects that detachment and closes the newly obtained media resources without upload.

### Browser capture/resampling

```text
getUserMedia
-> AudioContext actual sampleRate
-> repository AudioWorklet (channel average to mono Float32)
-> target-equivalent sample clock
-> dependency-free deterministic linear mono resampler to 16000 Hz
-> float clamp and signed little-endian PCM16
-> exact 44-byte WAV
-> raw authenticated upload
```

The encoder writes RIFF at 0, WAVE at 8, `fmt ` at 12, format code 1, mono, 16000, byte rate 32000, block align 2, 16 bits, `data` at 36, RIFF size `total-8`, and data size `samples*2`.

### Cleanup

- Success/error/discard closes the AudioWorklet graph and `AudioContext`, stops every media track, and drops buffers.
- Capture startup failure runs the same cleanup.
- `pointercancel` and `lostpointercapture` discard without upload.
- Zero-sample capture invokes the bounded discard callback.
- `pagehide` clears token, discards the recorder, closes WS, stops playback, and revokes the object URL.

## Upload bound and parser matrix

Pre-body order is token -> SID syntax/existence -> voice enabled -> active session -> concurrent upload -> Content-Type -> Content-Encoding -> Content-Length -> governance region. Only then is the request read. The evented reader accumulates at most `1,920,044` bytes and pauses/rejects at the first excess chunk.

The independent gateway parser accepts only:

```text
46..1920044 total bytes
RIFF/WAVE + fmt/data at canonical offsets
fmt length 16
PCM format 1
mono / 16000 Hz / 16 bit
byte rate 32000 / block align 2
data bytes == body bytes - 44
even data bytes
1..960000 samples
duration_ms = floor(samples * 1000 / 16000)
```

`voice.web-ptt-v0` Case D accepts a short WAV, 59.9 s, and exactly 60 s, and rejects 60 s + one sample, wrong RIFF/data magic, rate, channels, bits, PCM format, alignment, byte rate, truncated header/data, declared size mismatch, trailing bytes, odd data, and empty data.
