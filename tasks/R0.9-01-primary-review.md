## BLOCKER 1 — MiMo clearance profile is not closed

The gated contract fixes MiMo at:

```text
max_sensitivity: personal
residency: [region-restricted, global-ok]
regions: [cn]
```

Actual code passes `data_clearance` through the generic parser and schema, which still allow `secret`, `local-only`, and arbitrary regions. An owner could therefore configure the cloud ASR as secret-cleared and route data the slice was required to deny.

**Required repair**

Enforce the exact profile in both:

- `packages/config/src/schema.ts`
- `apps/gateway/src/speech-provider.ts`

Add negative tests for secret, local-only, missing/extra regions, and other non-exact combinations. Do not rely on JSON schema alone.

## BLOCKER 2 — Re-importing identical audio can declassify labels

`ArtifactRegistry.register()` derives the ID from content hash only and returns an existing record unchanged. `fairy voice import-audio` checks kind, MIME, and size, but not whether the existing record's labels cover the newly requested labels.

Failure path:

```text
same bytes already stored as public/global-ok input
→ import same bytes as secret/local-only
→ old weaker record returned
→ CLI accepts it
→ ASR routes using weaker labels
```

**Required repair**

Do not redesign artifact identity in this slice. Add a narrow fail-closed post-registration check:

- returned labels must cover the requested clamped labels;
- required region metadata must be preserved;
- otherwise the import fails and cannot invoke ASR.

Add tests for weaker-existing-label collision, same `kind:speech` collision, and proof that failed import cannot produce a cloud request.

Record the broader content-addressed artifact identity/label-merge issue as carry-in.

## BLOCKER 3 — ASR/turn concurrency and cancellation are unsafe

`voice.asr` can start while TurnRunner is active. Canonical `speech.asr.final` is emitted before `#submitVoiceFinalTranscript()`, whose eventual `#acceptTurnInput()` is the first place that checks whether a turn is already running.

Also:

```ts
this.#runner.cancel(sid) || await this.#speechProviders.cancelAsr(sid)
```

short-circuits. If TurnRunner cancellation succeeds, active ASR cancellation is never attempted.

**Required repair**

Introduce a deterministic per-session exclusion rule:

- reject `voice.asr` before staging/spawn when a turn is active;
- prevent a text turn from starting while ASR is active, or establish equivalent reservation semantics;
- always attempt runner and ASR cancellation separately, then combine results.

Add tests covering concurrent turn/ASR, dual cancellation, no final/turn/model after cancel, and unchanged normal success.

Do not implement barge-in.

## BLOCKER 4 — Owner-live procedure uses `dist`

The work report instructs:

```text
pnpm build
node apps/gateway/dist/...
node apps/cli/dist/...
```

This violates the pre-M5 source-first and single-`tsx` execution-world invariant.

**Required repair**

Replace the owner procedure with the repository's normal source-first launchers:

- `pnpm --filter @fairy/gateway start`
- `pnpm fairy ...`

Validate exact config/data-dir argument forwarding locally. Remove all plain-node-on-dist instructions. Owner live remains pending.

## CARRY-IN

### C1 — Artifact identity semantics

Content-only IDs cannot represent the same bytes under different kinds or independently strengthened labels. R0.9-01 should fail closed; a later neutral artifact-store task should decide identity and label-merge semantics.

### C2 — Provider count semantics

Connection/request counts are set after worker startup rather than transport-observed evidence. They are correct for tested success and zero-denial, but may overstate worker-local failures.

### C3 — Shutdown cancellation category

Coordinator shutdown can clear active state before the request classifies termination, producing failure rather than cancelled evidence. Cleanup remains bounded.
