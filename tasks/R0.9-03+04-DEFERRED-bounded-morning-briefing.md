## 4. Preserved functional scope

### 4.1 Plan mode

Required:

- read-only exploration;
- explicit plan artifact;
- ordered plan steps;
- expected tools;
- risks and permissions;
- time budget;
- token budget;
- cost budget;
- owner approve/reject;
- zero write-tool execution before approval.

Approval must be replayable and event-sourced.

### 4.2 Bounded execution loop

Required:

- maximum iterations;
- wall-clock deadline;
- token budget;
- cost budget;
- completion predicate;
- anomaly stop;
- explicit cancellation;
- visible budget state;
- every iteration uses the existing TurnRunner;
- no second agent loop;
- no in-memory-only truth.

### 4.3 Checkpoint and resume

Required:

- durable checkpoint;
- gateway crash/restart recovery;
- resume from canonical state;
- completed write steps are not repeated;
- incomplete step state is explicit;
- replay shows checkpoint and resume;
- failure/cancel/budget stop remain inspectable.

### 4.4 Morning Briefing

The only workflow in this slice is:

```text
Morning Briefing
```

Candidate inputs:

- current date;
- weather;
- calendar;
- bounded news/RSS or research summary;
- overnight workflow result when present.

Output:

- Markdown briefing artifact;
- canonical completion state;
- citations where research is used;
- optional MiniMax TTS.

It must fail soft when one data source is unavailable.

### 4.5 Scheduler-lite

Only:

- one persistent schedule;
- next trigger time;
- quiet hours;
- missed-run policy;
- gateway restart recovery;
- manual `run-now`;
- at most one concurrent run.

Scheduler is trigger-only.

It must not own workflow truth, plan state, checkpoints, or budget state.

### 4.6 Notification surface

Preferred minimum:

- Web UI inbox/status;
- optional local desktop notification only if low-risk and separately accepted.

No Telegram or multi-channel notification in this slice.

## 5. Canonical event discipline

Use the already-registered generic families:

```text
plan.*
loop.*
workflow.*
```

Do not add:

```text
execution.*
```

unless a future protocol gate proves the existing families cannot express the required state.

Allowed additive payload concepts, subject to fresh schema review:

```text
execution_id
root_execution_id
parent_execution_id
actor_kind
```

Canonical truth must express at least:

- workflow/plan created;
- plan proposed;
- plan approved/rejected;
- step started/completed/failed;
- checkpoint saved;
- budget updated/exhausted;
- workflow completed/cancelled/failed.

No browser/CLI-only state may be authoritative.

## 9. Future acceptance targets

The fresh task brief must include discriminating tests for at least:

### Approval

- write-tool count before approval = 0;
- reject ends workflow without writes;
- approved plan identity is immutable;
- stale approval cannot approve a replacement plan.

### Budgets

- exact iteration stop;
- exact wall-clock stop;
- token/cost exhaustion;
- no next iteration after exhaustion;
- bounded visible reason.

### Checkpoint/resume

- crash after completed write;
- resume does not repeat the write;
- crash during incomplete step;
- deterministic recovery classification;
- canonical replay.

### Cancellation/anomaly

- explicit cancel;
- cancel while tool pending;
- anomaly stop;
- no orphan process/step;
- terminal state exactly once.

### Scheduler

- persistent next run;
- restart recovery;
- quiet hours;
- missed-run policy;
- manual run-now;
- max concurrency one.

### Morning Briefing

- weather unavailable fail-soft;
- calendar unavailable fail-soft;
- research citation preservation;
- Markdown artifact;
- optional TTS;
- no implicit unapproved write.

### Architecture

- one TurnRunner;
- scheduler trigger-only;
- no second truth store;
- no `execution.*` family without gate;
- `server.ts` untouched or extraction-first gate satisfied.
