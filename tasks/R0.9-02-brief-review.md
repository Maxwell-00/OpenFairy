## Gate decisions

### D1 — Plan-file authority (the brief's §2 question): **option 2**

Both plan files are retained; **`docs/v0.9/OpenFairy-v0.9-final-tiered-plan-60s-amended.md` is the binding R0.9-02 overlay.** Rationale: append-only history is this project's discipline (DECISIONS law; deletes need explicit owner action), and the original plan remains the binding overlay for the slices it already governed. The `R0.9-06′` release gate should consolidate to a single plan file in the ledger pass — recorded here as a release-gate line item, not an implementation task.

### D2 — 60-second cap: **endorsed, math verified**

`960,000` samples × 2 B + 44 = `1,920,044` bytes ≈ 1.9 MB raw — 3.6× headroom under R0.9-01's 7,000,000 raw cap; predicted base64 ≈ 2.56 MB against the 10,000,000 whole-body constant. The owner's 60s choice is strictly more conservative than the plan gate's original 90s ruling; the amendment's derived constants (30s recommended / 50s warning / 55s countdown / 60s hard) are internally consistent. The "not a context-window claim" honesty line is appreciated.

### D3 — Browser WS auth via `?token=`: **accepted**

The browser `WebSocket` API cannot set headers; `?token=` is the existing M3-03-era authenticated-connect mechanism, not a new surface. Conditions already in the brief: token never echoed into events/ACKs/diagnostics/UI; invalid ⇒ `4401` with zero ops/sessions/artifacts; no new unauthenticated listener. The countersign adds one check: the gateway's request logging (if any) must not record query strings on the WS upgrade path.

### D4 — CSP `connect-src 'self' ws://127.0.0.1:*`: **accepted**

`'self'` does not reliably cover the `ws:` scheme across browsers; the wildcard port is confined to loopback on a page that is itself loopback-only. If the implementation can template the actual port without violating no-inline-script, narrower is welcome — Codex's choice, not a requirement.

### D5 — WAV-in-browser (plan-gate ruling) : **correctly landed**

`getUserMedia` + `AudioContext`/`AudioWorklet` + PCM→16 kHz mono 16-bit WAV with an exact 44-byte canonical header; `MediaRecorder` WebM explicitly forbidden; resampling required (constraints are hints, actual track settings authoritative). No transcoder exists anywhere. This discharges the plan-gate's WebM/WAV decision precisely.
