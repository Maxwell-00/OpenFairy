## Root cause

The reused/no-open branch owns no gateway child and opens no browser. It reaches:

```ts
if (!child) {
  await aborted(signal);
  return { ... };
}
```

The `aborted(signal)` promise only installs an AbortSignal event listener. That listener does not provide an active event-loop handle. The source-first CLI is awaiting `runDev()` at top level, so Node exits with an unsettled top-level-await warning instead of remaining alive until explicit Ctrl+C.

## Test gap

Existing Case J calls `runDev()` in the Vitest process and injects a browser callback that immediately aborts. It does not spawn the real source-first CLI in `--no-open` reused mode and therefore cannot detect spontaneous process exit.
