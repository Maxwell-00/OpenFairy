## 2. Findings

### BLOCKER

None.

### CARRY-IN — R0.9-01-C1: content-addressed artifact identity

Artifact IDs remain based on content bytes only. R0.9-01 correctly fails closed when an existing record has weaker labels, a different artifact kind, or missing/wrong region metadata.

A later neutral artifact-store task should decide:

- whether identity includes artifact kind;
- whether labels merge upward;
- how append-only registry updates represent strengthened labels;
- how the same bytes may exist under independently governed purposes.

This is not a R0.9-01 closure blocker.

### CARRY-IN — R0.9-01-C2: provider attempt versus observed transport counts

Current evidence fields are sufficient for the accepted success and zero-denial paths.

A later telemetry task may distinguish:

```text
provider_attempt_count
provider_connection_count
provider_request_count
```

so worker-local failures cannot overstate transport-observed activity.

This is not a R0.9-01 closure blocker.

### CARRY-IN — R0.9-01-C3: shutdown cancellation classification

Gateway shutdown remains bounded and cleans workers and staging roots. Some shutdown timing paths may classify termination as provider failure rather than cancelled evidence.

Do not reopen R0.9-01 for this. Carry it into later speech lifecycle work.

### NIT — R0.9-01-N1: replay evidence includes SQLite warning lines

`positive-replay.txt` and `denial-replay.txt` include Node's SQLite experimental warning before JSONL events because stderr was captured with stdout.

The files remain unambiguous and prove the required replay state. Future owner scripts should capture stderr separately or filter warnings before recording replay evidence.

### NIT — R0.9-01-N2: optional usage evidence was absent

The real provider ACK records `usage_seconds: null`.

Usage seconds were optional bounded evidence, not a success requirement. Provider identity, profile, model, auth, finish reason, provider request ID, request count, transcript, canonical events, and downstream turn evidence are all present.

### NIT — R0.9-01-N3: owner summary is intentionally pre-review

`owner-live-summary.json` records `owner_live_closed: false`.

That is correct for owner-generated evidence: the owner run does not self-close the task. This final primary review supplies the closure decision.
