### 5.1 CLI

```text

```

### 5.2 Structured report

Replace the text-only internal result with a structured, deterministic report while retaining human-readable rendering.

Required conceptual shape:

```ts
interface DoctorCheck {
  id: string;
  status: "pass" | "warn" | "fail";
  summary: string;
  remediation?: string;
}

interface DoctorReport {
  kind: "fairy.doctor.report";
  v: 1;
  ok: boolean;
  checks: readonly DoctorCheck[];
  lines: readonly string[];
}
```

Exact TypeScript organization may differ, but these semantics are required.

Rules:

- fixed check order;
- stable check IDs;
- `ok === true` only when no check is `fail`;
- warnings do not force a nonzero exit;
- text mode is readable and bounded;
- JSON mode prints exactly one JSON object to stdout;
- JSON mode contains no ANSI sequences;
- failures produce exit code `1`;
- success with warnings produces exit code `0`.

### 5.3 Required check registry

Use these stable check IDs, in this order:

```text
runtime.node
runtime.pnpm
runtime.python
config.load
config.main-role
config.gateway-auth
config.speech-asr
config.speech-tts
storage.data-dir
storage.artifacts
storage.sessions
network.gateway-port
runtime.gateway-health
web.assets
optional.container-runtime
optional.research
```

Each check must return only `pass`, `warn`, or `fail`.

### 5.4 Check semantics

#### `runtime.node`

- PASS for Node `>=22`;
- FAIL otherwise;
- include version, never executable path.

#### `runtime.pnpm`

- PASS when pnpm is executable and returns a version;
- FAIL otherwise;
- external process probe deadline `<=2,000 ms`.

#### `runtime.python`

The doctor must use the same Python candidate order and the same `>=3.11` validation law as speech workers.

Do not copy a divergent implementation.

Preferred implementation:

- extract only interpreter candidate discovery and version-floor validation into an owned provider-neutral module under `packages/voice`;
- have both `SpeechWorkerProcess` and doctor import it;
- preserve production behavior and existing worker tests;
- keep `FAIRY_TEST_PYTHON` test/CI-only exactly as today;
- do not expose arbitrary interpreter paths through YAML or CLI.

Semantics:

- PASS when the production interpreter is discoverable and `>=3.11`;
- FAIL when ASR/TTS providers are configured but no supported interpreter is available;
- WARN when speech providers are absent and Python is unavailable;
- do not print an absolute interpreter path in the default text report;
- JSON may include only source class and version, not credential or worker details.

#### `config.load`

- load the same effective config as gateway/CLI;
- respect `--config`, `FAIRY_CONFIG`, user config, workspace config, and session overrides in the existing precedence order;
- PASS when parse/schema validation succeeds;
- FAIL with bounded issue paths and remediation otherwise;
- never print expanded secret values;
- source names are allowed;
- full paths may be shown only in local text detail, not committed deterministic fixtures.

#### `config.main-role`

- FAIL when `roles.main` is missing or binds an unknown model;
- PASS when the existing model-gateway parse contract accepts it;
- check model credential reference presence when configured;
- do not call the model endpoint;
- a local/no-key model may PASS without an API-key reference.

#### `config.gateway-auth`

- FAIL for missing/empty auth;
- FAIL for the unchanged default `dev-token`;
- PASS for a resolvable `secret://` reference;
- WARN for a non-default literal token;
- never print, hash, partially reveal, measure, or echo the token;
- remediation may print the secret-reference name, never its value.

#### `config.speech-asr`

- use existing speech-provider parsing;
- PASS when the ASR role resolves to the closed MiMo provider contract and its secret reference is present;
- validate ordinary MiMo pay-as-you-go credential class `sk-*` without printing it;
- FAIL for missing role/provider/credential or wrong credential class;
- do not connect to MiMo.

#### `config.speech-tts`

- use existing speech-provider parsing;
- PASS when the TTS role resolves to the closed MiniMax provider contract and its secret reference is non-empty;
- FAIL for missing role/provider/credential;
- do not connect to MiniMax.

#### `storage.data-dir`

Resolve the same effective data directory as the gateway.

- PASS when it exists and is a writable directory;
- PASS when it is absent but its nearest existing parent is writable;
- FAIL for file/directory collision, inaccessible parent, or failed bounded write probe;
- any write probe must use a random temporary file and remove it in `finally`;
- do not initialize unrelated storage;
- do not leave residue.

#### `storage.artifacts`

- PASS when the artifact root/registry is absent and can be initialized by normal runtime;
- PASS when existing registry state is readable through repository-owned artifact APIs;
- FAIL when an existing registry is malformed, escaped, or unreadable;
- do not enumerate artifact contents in output;
- do not print artifact paths or hashes.

#### `storage.sessions`

- PASS when the sessions root is absent and can be initialized normally;
- PASS when the existing root is a readable directory;
- FAIL for file collision or unreadable root;
- do not parse every historical session;
- do not print transcript/event content.

#### `network.gateway-port`

Against the effective loopback port:

- PASS `available` when a bounded bind probe succeeds;
- PASS `fairy-running` when `/health` returns the expected Fairy health shape;
- FAIL `occupied` when another process owns the port and Fairy health is absent/invalid;
- no host override;
- loopback only;
- bounded deadline `<=1,000 ms`.

#### `runtime.gateway-health`

- PASS when an existing Fairy gateway returns expected health;
- WARN when the port is available and gateway is not running;
- FAIL when the port is occupied by a non-Fairy or unhealthy service;
- no session/provider operation.

#### `web.assets`

Validate the exact repository-owned source assets:

```text
apps/web/index.html
apps/web/styles.css
apps/web/app.js
apps/web/recorder.js
apps/web/wav.js
apps/web/audio-worklet.js
```

- PASS when all are regular readable files;
- FAIL when missing, symlinked, unreadable, or replaced by directories;
- do not add a general static-directory scan.

#### `optional.container-runtime`

- PASS when Docker or Podman version probe succeeds;
- WARN when absent;
- never FAIL Tier-1 launch solely because it is absent;
- bounded external probe.

#### `optional.research`

- PASS when a research engine is configured and accepted by existing config;
- WARN when research is unavailable or only deterministic/mock capability is configured;
- never issue a search/fetch;
- do not block normal voice demo readiness.

### 6.1 CLI


### 6.2 Preflight

Before spawning anything:

1. run the same doctor registry with the same effective overrides;
2. render bounded failures;
3. refuse launch when any required check fails;
4. spawn zero child processes on failure;
5. open zero browser processes on failure.

Warnings are printed but do not block launch.

Doctor and dev must not diverge in config, secret, Python, port, data-dir, or Web-asset checks.

### 6.3 Gateway ownership modes

The effective port has exactly three outcomes.

#### Mode A — available port

- spawn the existing source-first gateway launcher;
- use `scripts/start-gateway.mjs`;
- pass only validated `--config`, `--data-dir`, and `--port` arguments;
- use `shell:false`;
- do not put tokens or provider credentials in argv;
- mark the child as owned by the dev launcher.

#### Mode B — healthy Fairy gateway already running

- reuse it;
- spawn no gateway child;
- print `Gateway: REUSED`;
- do not claim ownership;
- on Ctrl+C, exit the dev launcher without killing the pre-existing gateway.

#### Mode C — occupied non-Fairy/unhealthy port

- fail before browser open;
- spawn no gateway child;
- print bounded remediation;
- do not probe arbitrary endpoints.

### 6.4 Startup readiness

For an owned child:

- poll only `http://127.0.0.1:<port>/health`;
- code-owned total startup deadline `15,000 ms`;
- bounded poll interval;
- browser open only after expected health succeeds;
- if child exits before health, propagate a nonzero result;
- if deadline expires, terminate the owned child and fail;
- never inspect or log WebSocket URLs containing tokens.

### 6.5 Browser open

After health succeeds, use:

```text
http://127.0.0.1:<port>/web/
```

Rules:

- no token in URL;
- URL is code-constructed from validated integer port;
- Windows launch is supported;
- `--no-open` skips browser launch;
- browser-open failure is WARN, not a reason to kill a healthy gateway;
- no new runtime dependency;
- no user-controlled executable path;
- no arbitrary URL option.

Print the URL so the user can open it manually.

### 6.6 Console contract

Required bounded text:

```text
Fairy dev
Doctor: PASS | PASS WITH WARNINGS
Gateway: STARTED | REUSED
Health: http://127.0.0.1:<port>/health
Web: http://127.0.0.1:<port>/web/
Auth: enter the configured gateway token in the Web UI; token is not printed
Press Ctrl+C to stop this launcher
```

Do not print:

- gateway token;
- provider credential;
- WebSocket query URL;
- provider endpoint response;
- child environment;
- full config object.

### 6.7 Ctrl+C and cleanup

For an owned gateway:

1. forward graceful shutdown;
2. wait a code-owned bounded deadline;
3. force termination only if needed;
4. wait for exit;
5. exit with the appropriate code;
6. leave the port free;
7. leave no child/grandchild process;
8. leave no launcher temp file.

For a reused gateway:

- do not signal it;
- do not close it;
- exit only the launcher.

Handle `SIGINT` and `SIGTERM`.

Windows behavior is mandatory.

### 6.8 No hidden side effects

`fairy dev` must not:

- install dependencies;
- write provider keys;
- create a config file;
- modify the user's config;
- call a provider;
- create a session;
- create an artifact;
- submit a turn;
- modify canonical history;
- silently switch data directories;
- use a remote bind.

Normal gateway initialization of its configured data directory after successful launch remains allowed.
