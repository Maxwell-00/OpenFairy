## 9. Artifact validation and staging lifecycle

Accepted records are exactly `kind: input` and `mime: audio/wav | audio/mpeg`. Validation rejects invalid artifact ID, missing record/file, speech/perception kind, WebM/Opus/aliases, empty/oversize data, relative/traversal/UNC/drive-relative/ADS paths, path escape, symlink, non-regular file, magic mismatch, size/hash mutation, and TOCTOU changes. A file handle is used for stable before/opened/after identity and metadata checks.

Allowed lifecycle:

1. resolve content-addressed record;
2. validate record/path/regular-file/size/hash/magic;
3. derive effective labels;
4. apply provider clearance and egress;
5. enforce predicted request cap;
6. resolve pay-go credential;
7. create private OS temp root;
8. write `asr-input.bin.partial`, sync, rename to `asr-input.bin`, then revalidate size/hash/magic;
9. launch one worker and send metadata-only `asr.request`;
10. terminate/shutdown and recursively remove the root on success, error, cancel, timeout, crash, malformed output, and gateway shutdown.

Under-cleared requests stop before step 7.

## 17. Cancellation/crash/timeout lifecycle

ASR lifecycle is one bounded worker per request. A gateway-owned per-session reservation is acquired synchronously before the first provider or turn await and has the exact bounded state `{ kind, token, cancelled }`: an active turn rejects `voice.asr` before staging/spawn, and an active ASR rejects ordinary `turn.input` before canonical input/model execution. `turn.cancel` first marks an ASR reservation cancelled, then independently invokes both `TurnRunner.cancel(sid)` and `SpeechProviderCoordinator.cancelAsr(sid)`; the ACK is true if the reservation mark, runner cancellation, or coordinator cancellation succeeds, with no `||` short circuit.

The exact-token reservation supplies `isCancelled()` to the coordinator. Cancellation is checked after artifact validation, before staging, after temporary-root creation, after staging/before worker construction, after worker startup/before the provider request, after provider completion, and again after worker shutdown/temp cleanup before an accepted transcript can return. A detected cancellation starts no later phase, performs no retry, terminates any existing worker, returns only bounded `asr_cancelled`/`cancelled` evidence, and reaches the same cleanup `finally`. The coordinator removes `#activeAsr` only when the map still holds the exact active-operation object.

After an accepted provider result, `server.ts` checks that the same reservation token still owns the session and `cancelled` is false immediately before canonical `speech.asr.final`; it checks again after canonical persistence and before the sole `#submitVoiceFinalTranscript` → `#acceptTurnInput` path. The outer operation cleanup removes the session reservation only when its exact token still owns the entry, so an older operation cannot delete a later reservation. Gateway shutdown, crash, malformed stdout, response failure, read/total/supervisor timeout, and partial/truncated output retain the existing bounded cleanup law.

Two code-only Promise barriers make the formerly uncovered windows deterministic without sleeps: `beforeAsrCoordinator` pauses after the gateway reservation and before coordinator validation/staging; `beforeAsrCanonicalFinal` pauses after a valid provider result and before canonical emission. `MinimalGateway` rejects either hook unless `NODE_ENV=test` or `CI=true`; neither hook is present in YAML, CLI, configuration schema, or environment parsing.
