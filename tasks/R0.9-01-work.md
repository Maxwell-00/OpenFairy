## 9. Artifact validation and staging lifecycle

Accepted records are exactly `kind: input` and `mime: audio/wav | audio/mpeg`. Validation rejects invalid artifact ID, missing record/file, speech/perception kind, WebM/Opus/aliases, empty/oversize data, relative/traversal/UNC/drive-relative/ADS paths, path escape, symlink, non-regular file, magic mismatch, size/hash mutation, and TOCTOU changes. A file handle is used for stable before/opened/after identity and metadata checks.

Allowed lifecycle:

1. resolve content-addressed record;
2. validate record/path/regular-file/size/hash/magic;
3. derive effective labels;
4. apply provider clearance and egress;
5. enforce predicted request cap;
6. resolve pay-go credential;
7. create private OS temp root;
8. write `asr-input.bin.partial`, sync, rename to `asr-input.bin`, then revalidate size/hash/magic;
9. launch one worker and send metadata-only `asr.request`;
10. terminate/shutdown and recursively remove the root on success, error, cancel, timeout, crash, malformed output, and gateway shutdown.

Under-cleared requests stop before step 7.

## 11. Size calculation and deadlines

Limits:

| Limit | Value |
|---|---:|
| raw input | 7,000,000 bytes |
| full predicted JSON request | 10,000,000 decimal bytes |
| provider response | 1,048,576 bytes |
| normalized transcript | 20,000 characters |
| worker stdout line | 262,144 bytes |
| captured stderr | 8,192 bytes |

Prediction is `4 * ceil(raw_bytes / 3)` plus the exact UTF-8 byte count of the fixed JSON envelope and exact `data:<MIME>;base64,` prefix. Gateway rejects before staging/connection when the predicted body is too large; the worker recomputes and enforces the same whole-body constant before opening HTTP.

Deadlines:

| Layer | Connect | Read | Total/request |
|---|---:|---:|---:|
| MiMo HTTP | 10,000 ms | 60,000 ms | 75,000 ms |
| gateway supervisor | n/a | n/a | 80,000 ms |
| cancellation | n/a | n/a | existing 2,000 ms termination bound |

## 17. Cancellation/crash/timeout lifecycle

ASR lifecycle is one bounded worker per request. `turn.cancel` also cancels the active same-session provider operation. The coordinator terminates the blocking stdlib child, returns a bounded cancelled ACK, emits no ASR final/turn/model request, and deletes the staging root. A duplicate operation is rejected before staging when ownership already exists; the later atomic check covers concurrent entry, and only the invocation that inserted the active entry may delete it. Gateway shutdown marks all active operations cancelled, terminates children, waits for operations, then flushes existing canonical state. Crash, malformed stdout, response failure, read/total/supervisor timeout, and partial/truncated output use the same cleanup `finally` path.
