# REMAINING BLOCKER — Cancel-before-final still has unobservable windows

## Contract

Any cancel before accepted `asr.final` must produce:

- no canonical ASR final;
- no `turn.input`;
- no model request;
- no worker/temp residue.

## Current state

The session reservation records only:

```ts
{ kind: "asr" | "turn", token: symbol }
```

`turn.cancel` does not mark the reservation itself as cancelled. It only calls:

```ts
runner.cancel(sid)
await speechProviders.cancelAsr(sid)
```

The coordinator does not add the ASR operation to `#activeAsr` until after:

- artifact validation;
- clearance and egress;
- request-size checks;
- credential resolution;
- temp-root creation;
- artifact staging;
- worker construction.

Therefore this sequence remains possible:

```text
voice.asr reserves session
→ turn.cancel arrives during validation/staging
→ runner.cancel = false
→ coordinator.cancelAsr = false because #activeAsr is not registered yet
→ cancel ACK reports false
→ ASR continues and may emit final/turn/model
```

A second window exists after `SpeechProviderCoordinator.runAsr()` returns success. Its `finally` has already removed `#activeAsr`, but `server.ts` has not yet emitted `speech.asr.final`. A cancel in that interval again sees no runner and no active coordinator operation, after which the final may still be emitted.

The current test waits until the fake provider has received one request before cancelling. It does not cover either window.

## Required narrow repair

Make the session reservation cancellation-aware.

Accepted shape:

```ts
{ kind, token, cancelled }
```

or an equivalent `AbortController`.

Requirements:

1. `turn.cancel` marks an active ASR session reservation cancelled even if no worker is registered yet.
2. Coordinator cancellation can be requested before worker construction:
   - register an operation shell at entry, with optional worker; or
   - accept an AbortSignal/cancellation callback and check it before staging, before spawn, and after provider completion.
3. `#runVoiceAsr()` checks reservation cancellation immediately before emitting canonical `speech.asr.final`.
4. A cancellation after provider success but before canonical final discards the transcript and emits no final/turn/model.
5. Cancel ACK returns `cancelled:true` when the ASR reservation was marked, even if no worker existed yet.
6. Cleanup remains bounded.
7. Do not implement barge-in.

## Required tests

### A. Immediate cancel

```text
voice.asr
→ cancel before staging/worker registration
```

Assert:

- cancel ACK true;
- no final/turn/model;
- ideally zero spawn/staged bytes/connection when cancellation wins before those phases;
- no residue.

Use a code-gated test pause before coordinator worker registration rather than timing-dependent sleeps.

### B. Post-provider/pre-canonical cancel

Pause after provider success is accepted by the coordinator but before `speech.asr.final` emission, then cancel.

Assert:

- cancel ACK true;
- no canonical final;
- no turn/model request;
- no residue.

### C. Existing cases remain green

- active turn → ASR rejected zero-I/O;
- active ASR → text turn rejected;
- cancellation during blocking provider request;
- ordinary success exactly one final/turn/model.
