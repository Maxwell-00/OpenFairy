## Verdict

**ADOPT plan 2's structure (v0.9 → v0.9.1) WITH a tiered cut-line for the 1–2 week window.** The overlay-track approach is approved: `R0.9-*` is a release overlay, not a milestone renumbering; every §3 invariant stays a hard gate; M3-06 registers as `DEFERRED TO FULL M3 / v1.0` (brief + gate record + pinned revision remain valid, nothing cancelled).

**Timeline honesty is this gate's main intervention.** Plan 1's own estimate is 8–14 focused days for full v0.9; plan 2 adds 3–6 more. A 1–2 calendar-week window does not hold 11–20 focused days. Therefore:

### Tier 1 — the interview demo core (target ~5–7 focused days; do these, in order)

- **R0.9-01** cloud file ASR (see decisions below — reuse the entire M3-05 provider-worker pattern; the speech-coordinator carry-in lands here).
- **R0.9-02** localhost web push-to-talk + MP3 playback.
- **R0.9-05′** trimmed hardening: `doctor` + one-command dev start + demo script/README. No installer polish — interviewers watch demos, they don't install.
- **R0.9-06′** light release gate + `docs/v0.9-deferrals.md` ledger.

The demo story this buys: **full voice loop (speak → governed agent → spoken answer) on top of an already-built trust stack** — live `fairy replay` of the session, a route-denial on secret content, memory hold/write, research with citations, persona. The differentiators (labels, zero-byte denial, replayable everything, countersigned reviews) cost **zero new work** — they are the demo script, not the sprint. Few interview candidates can show a labeled, replayable, review-disciplined agent runtime; that story is already on disk.

### Tier 2 — only if days remain

- **R0.9-03+04 merged** (plan 1 already offers the merge): ONE approval-gated, budget-bounded, resumable Morning Briefing on a minimal event-shaped plan/loop. **With plan 2's subagent-ready constraints accepted** (see Q-block) — they cost ~0.5–1 day now and 7–12 days retrofitted later, by plan 2's own math.

### v0.9.1 (SA-01..04) — after the interviews, not in the window

Subagents-lite is well-shaped (agent-as-tool, depth 1, concurrency 2, Research+Critic only, children memory-write-less) but it is not demo-critical, and the honest interview line if asked is stronger anyway: "the execution contract is subagent-ready — here is the gated design." Showing the countersigned design docs IS the flex.
